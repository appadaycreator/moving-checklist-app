## 🔴 P1: 必ず実装すべき改善

### 1. チェックリストのリセット・クリア機能追加
**目的**: ユーザーが新しい引越し計画を開始する際、前回のチェック状態を簡単にクリアできるようにする  
**現状**: 履歴から選んでも、前回のチェック状態が残ったままで新規スタートできない  
**仕様**:
- 新規引越しデータを入力して「チェックリストを作成」ボタンをクリック時に、前回のチェック状態をリセット
- または「チェックリストを作成」ボタンの直下に「🗑 チェック状態をリセット」ボタンを追加
- クリック時: 確認ダイアログを表示 → localStorage の `moving-checklist-app-checks` をクリア

**実装タスク**:
1. リセット機能を `generate()` 関数に統合 OR 独立したボタン `resetChecklist()` を実装
2. ボタンスタイル: グレー背景・12px フォント・max-width対応
3. 確認ダイアログ: シンプルな `confirm()` で OK

**DoD**:
- リセット後、チェックボックスが全てアンチェック状態に戻る
- 過去のチェック履歴には影響しない
- モバイル・PCでボタンが見やすく・タップしやすい（44px min-height）

---

### 2. カスタムタスク追加機能（ユーザー任意タスク入力）
**目的**: プリセットのチェックリストに加えて、ユーザーが自分独自の引越しタスクを追加できるようにする  
**現状**: 固定のチェックリストのみで、カスタムタスク追加の手段がない  
**仕様**:
- チェックリスト表示後、チェックリストの下部に「➕ カスタムタスクを追加」セクションを表示
- テキスト入力欄 + 「追加」ボタン
- 追加されたタスクは「📝 その他」セクションに表示
- カスタムタスク情報は localStorage に `moving-checklist-app-custom-tasks` で保存
- チェック状態も追跡

**実装タスク**:
1. `buildOutput()` 関数で、チェックリスト生成後に カスタムタスクセクションを追加
2. 入力フォーム: `<input type="text" id="custom-task-input" placeholder="例：駐車場の解約手続き">`
3. 追加ボタン onClick: 新しいタスク オブジェクトを作成 → localStorage保存 → UIリフレッシュ
4. カスタムタスク削除機能: 各タスク横に🗑ボタン

**DoD**:
- テキスト入力後、「追加」ボタンで瞬時にリストに反映
- カスタムタスクもチェック状態を保持
- ページリロード後も追加したタスクが復元される
- キーボードの Enter キーでも追加可能

---

### 3. 形式的バリデーション改善（UX向上）
**目的**: フォーム未入力時のエラー表示を改善し、より親切なガイダンスを提供する  
**現状**: エラーが表示されるが、どの項目が未入力か判断しづらい  
**仕様**:
- 各入力欄の未入力時に、フォーカス時またはボタンクリック時に **対象項目をハイライト**（赤色ボーダー）
- エラーメッセージを複数行で表示（1行ずつ）
- エラー表示時、自動的に最初の未入力項目にスクロール・フォーカス

**実装タスク**:
1. `getInputs()` 関数で、エラー時に各フォーム要素に CSS クラス追加（例：`input.classList.add('input-error')`）
2. CSS: `.input-error { border-color: #dc2626 !important; background: #fee2e2; }`
3. エラーメッセージを `<ul>` で複数行表示
4. `errDiv.scrollIntoView()` + 最初のエラー項目へ `focus()`

**DoD**:
- エラー項目が赤色でハイライトされ、視認性UP
- ユーザーが直感的に修正対象を理解
- エラーメッセージは改行で見やすく

---

### 4. 進捗グラフの自動非表示 & 初回表示改善
**目的**: ページロード初期状態で不要なグラフを非表示にし、ページサイズを軽くする  
**現状**: チェックリスト未生成状態でも進捗セクション（グラフ含）が存在し、ページが重い  
**仕様**:
- 初期状態: `charts-section` は非表示（`display: none` のまま）
- チェックリスト生成時のみ表示
- チェック数が0件の場合も、グラフを生成しない

**実装タスク**:
1. `updateProgress()` 関数で、`total === 0` 時にも `charts-section` を非表示に
2. `generateWithData()` 時にのみグラフ生成処理を実行
3. メモリリーク対策: グラフ生成時の `Chart.js` インスタンスを確実に破棄

**DoD**:
- ページロード時の初期レンダリング速度UP
- チェックリスト生成直後にグラフが瞬時に表示される

---

### 5. 日付入力の初期値・最小値改善
**目的**: ユーザーが日付入力時に迷わないよう、デフォルト値・制約を明確にする  
**現状**: 日付フィールドに最小値は設定されているが、プレースホルダーがない  
**仕様**:
- 引越し予定日: デフォルト値を「今日から7日後」に設定
- 目標完了日: デフォルト値を「引越し日の7日前」に設定（オプション）
- 最小値: 両方とも「今日」以降に制限（既実装）

**実装タスク**:
1. DOMContentLoaded イベント内で、デフォルト日付を設定
   ```js
   const today = new Date();
   const sevenDaysLater = new Date(today);
   sevenDaysLater.setDate(sevenDaysLater.getDate() + 7);
   document.getElementById('movedate').value = sevenDaysLater.toISOString().split('T')[0];
   ```
2. `<small>` タグで「推奨: 引越し日は今日から2週間以上先の日付を推奨」と表記

**DoD**:
- ユーザーが日付を迷わず入力可能
- 初期値があると「試し使い」のハードルが低下

### 6. カテゴリ別進捗表示
**目的**: ユーザーが「役所手続き」「荷造り」など、カテゴリ単位でどれだけ完了しているか把握できるようにする  
**現状**: 全体進捗率のみで、カテゴリ別の進捗がわからない  
**仕様**:
- 進捗セクション内に「📂 カテゴリ別進捗」アコーディオンを追加
- 各カテゴリ（8週前～引越し後）ごとに、完了数/総数を表示
- プログレスバーでカテゴリ別進捗を可視化

**実装タスク**:
1. `buildOutput()` で、カテゴリ別の完了数を集計
2. HTML: アコーディオン UI（onclick で展開/折畳）
3. CSS グリッド: 各カテゴリにプログレスバー

**DoD**:
- 「役所手続き」など各セクションの完了率が表示される
- アコーディオンで開閉可能
- 初期状態は折りたたまれている（ページ軽量化）

### 7. A8アフィリエイト・関連商品の文脈的配置
**目的**: チェックリスト生成後、ユーザーの選択内容（家族構成・移動距離）に基づいて、関連アフィリエイト商品を動的に表示する  
**現状**: すべてのユーザーに同じアフィリエイトリンク（アールクリーニング、楽天、Amazon）を表示  
**仕様**:
- ファミリー（子あり）→「ベビー用品・子供向け梱包材」推奨
- 遠距離移動 → 「引越し保険・高額補償パック」推奨
- 一人暮らし → 「レンタルボックス・オークション出品代行」推奨

**実装タスク**:
1. `buildOutput()` で、`inputs.family` / `inputs.movetype` に基づいてアフィリエイトリンク集合を変更
2. 既存の A8.net ビーコンコードは **必ず保持**（削除禁止）
3. 新規セクション: チェックリスト直後に「💡 あなたにおすすめの引越しサービス」を挿入

**DoD**:
- ファミリー向けの商品が動的に表示される
- A8.net ビーコン（px.a8.net）は削除されない
- CTR/コンバージョン向上

### 8. スマートフォン向け UI 最適化（タップターゲット・レイアウト）
**目的**: モバイルでの操作性を向上させ、誤タップを減らす  
**現状**: 一部のボタン・チェックボックスが小さく、タップしにくい可能性がある  
**仕様**:
- チェックボックス: 最小幅 44px × 44px（WCAG推奨）
- ボタン: min-height 48px（既実装だが再確認）
- フォーム入力欄: min-height 44px（既実装）
- カテゴリラベル: フォントサイズを 13px → 14px に拡大

**実装タスク**:
1. CSS メディアクエリで `@media (max-width: 480px)` 内に以下を追加:
   ```css
   input[type="checkbox"] { width: 20px; height: 20px; min-width: 44px; min-height: 44px; }
   label { padding: 8px 0; }
   ```
2. チェックボックス周囲に適切な padding を確保

**DoD**:
- モバイル上でチェックボックスが44px以上のタップエリアを確保
- 誤タップが減少

---

### 9. 印刷・PDF出力時の改善
**目的**: 印刷版チェックリストの見栄えと機能性を向上させる  
**現状**: `@media print` で不要要素は非表示にしているが、チェック済みタスクの視認性が低い  
**仕様**:
- 印刷時に、チェック済みタスクを **打ち消し線** で表示
- または チェック済みを **グレー + 薄く** 表示
- 印刷時、日付・家族構成・タスク数を ヘッダーに表示（印刷者識別用）

**実装タスク**:
1. CSS:
   ```css
   @media print {
     input[type="checkbox"]:checked + label { 
       text-decoration: line-through; 
       opacity: 0.6; 
     }
     body::before { content: "引越しDate: "; }
   }
   ```

**DoD**:
- 印刷版がより見やすく、実用的になる
- チェック済みタスクが視覚的に区別できる

---

### 10. キーボードナビゲーション・アクセシビリティ改善
**目的**: キーボード操作のみでアプリを利用でき、スクリーンリーダー対応を強化する  
**現状**: マウス操作が主体で、キーボードナビゲーションの確認が不十分  
**仕様**:
- Tab キーで フォーム → ボタン → チェックボックス を順に移動
- Enter キーで「チェックリストを作成」実行
- リセットボタンを追加時も Tab 対応
- Escape キーでモーダルを閉じる（履歴選択時）

**実装タスク**:
1. `restorePreviousData()` 内のモーダルに Escape キーリスナー追加
2. フォーム要素に自動 tab-index（HTML の順序で自動）
3. 各ボタンに aria-label 確認

**DoD**:
- キーボード操作のみでアプリ全体を操作可能
- Escape キーでモーダルが閉じられる
